# Changelog

Все заметные изменения проекта. Формат — [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),

## [1.2.0] — 2026-06-10

Харденинг безопасности и надёжности; чистка репозитория.

### Исправлено
- Mini App: переключение вкладок больше не пересоздаёт экран и не сбрасывает
  загруженные данные в пустой «скелет». Вкладки остаются в памяти, переключение
  мгновенное, обновление данных идёт в фоне без мигания.
- Админ-панель: список профилей не очищается при обновлении (после
  блокировки/разблокировки) — старые строки остаются до прихода новых.

### Безопасность
- Constant-time сравнение всех внутренних токенов (`crypto/subtle`).
- Раздельные внутренние токены на каждый сервис: `INTERNAL_TOKEN_CONNECT`,
  `INTERNAL_TOKEN_BOT` (legacy `ADMIN_TOKEN` — fallback).
- Rate limiting по IP: `/auth/token` (60/мин), `/auth/key` (20/мин) — защита от
  перебора ключей.
- Отзыв JWT: на каждом запросе проверяется владелец — заблокированные/удалённые
  пользователи отсекаются, несмотря на 30-дневный срок токена.
- Ротация JWT-секрета без разлогина: `JWT_SECRET_PREVIOUS` принимается verify-only.
- xray gRPC API (`:10085`) привязан к docker-бриджу вместо `0.0.0.0`; установщик
  сам применяет правила UFW.
- nginx: HSTS, `X-Content-Type-Options`, CSP `frame-ancestors`, `client_max_body_size`.
- `absolute_redirect off` — редирект `/` → `/v2/` больше не светит внутренний `:8443`.
- nginx: access-логи отключены на веб-вхостах и SNI-stream — сервис не хранит
  пер-запросные записи IP/SNI/вызовов API (приватность VPN).
- Убран захардкоженный IP сервера из дефолтов (`SERVER_IP` обязателен).
- `setup-deploy.sh` больше не печатает приватный ключ в stdout.

### Надёжность
- Фикс TOCTOU в проверке лимита устройств (блокировка строки пользователя в транзакции).
- Уникальный индекс устройств + `ON CONFLICT DO NOTHING` — гонка не плодит дубликаты.
- Graceful shutdown в `connect` и `awg-server`.
- Лимиты CPU/RAM на все контейнеры.
- Обработка ранее проглатываемых ошибок JSON-декодирования.

### Тесты и CI
- Юнит-тесты: парсер User-Agent, проверка `initData` (HMAC), учёт трафика, ротация JWT.
- `go test ./...` для всех Go-модулей в CI.

### Инфраструктура
- Автодеплой переведён на read-only GitHub deploy key (SSH через порт 443) —
  больше не зависит от протухающих токенов в git-remote.

### Удалено
- Незавершённые заглушки sub-config (эндпоинты `/configs/{id}/subconfig` и таблица
  `subconfigs`) — функция не использовалась.

## [1.1.0] — 2026-05-29

- Вкладка «Настройки»: профиль, устройства (переименование/блок/удаление), лимит
  устройств, язык, сброс подписки, удаление аккаунта, «О сервисе».
- Админ-панель в Mini App: ключи, профили, трафик (всего/за сегодня), домены.
- Автологин, учёт устройств по HWID, enforcement лимита устройств в `connect`.

## [1.0.0] — 2026-05

- VLESS + REALITY (Vision/TCP `:43000`, XHTTP `:43001`) и AmneziaWG (`:51820`).
- Mini App (React + TS + Tailwind, Claude-style), бот на grammY.
- API на Go: auth по Telegram `initData` → JWT, ключи доступа, конфиги, подписки,
  per-device провижининг, cron-учёт трафика и метрик сервера.
- Docker Compose, nginx SNI-роутер, ежедневные бэкапы БД.
